# 多 Agent 协作测试系统 - 使用指南

## 🎯 快速开始

### 第一步：启动应用

```bash
cd <project-root>/web
npm run dev
```

确保应用运行在 `http://localhost:5173`

---

### 第二步：运行测试

**方式 1：完整测试流程（推荐）**
```bash
npm run test
```
这会运行所有测试并生成人类可读的报告。

**方式 2：有头模式（可见浏览器，方便调试）**
```bash
npm run test:e2e:headed
```

**方式 3：UI 模式（交互式调试）**
```bash
npm run test:e2e:ui
```

## 🤖 多 Agent 协作流程

### 流程说明

```
┌─────────────────────────────────────────────────────────┐
│  1. 前端测试 Agent 自动运行 8 个测试用例                    │
│     - Playwright 执行浏览器自动化测试                      │
│     - 收集测试结果和截图                                  │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  2. 生成测试报告                                         │
│     - 解析测试结果                                      │
│     - 识别问题类型（前端/后端）                          │
│     - 生成 JSON 和人类可读报告                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  3. 等待用户确认                                         │
│     - 显示测试报告                                      │
│     - 询问是否执行修复                                  │
│     ⚠️ 此处需要您确认才能继续                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼ 您确认"是"
┌─────────────────────────────────────────────────────────┐
│  4. 调度对应 Agent 执行修复                               │
│     - 前端问题 → 前端开发 Agent                          │
│     - 后端问题 → 后端开发 Agent                          │
│     - 分析代码，生成修复方案                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  5. 修复方案确认                                         │
│     - 显示修复方案（修改文件、代码变更）                 │
│     ⚠️ 此处需要您确认才能继续                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼ 您确认"是"
┌─────────────────────────────────────────────────────────┐
│  6. 执行修复并验证                                       │
│     - 修改代码                                          │
│     - 重新运行测试                                      │
│     - 生成验证报告                                      │
└─────────────────────────────────────────────────────────┘
```

## 🔍 问题定位规则

测试系统会自动识别问题类型：

| 错误模式 | 判定为 | 处理 Agent |
|----------|--------|-----------|
| selector not found | 前端 | 前端开发 |
| element not visible | 前端 | 前端开发 |
| click timeout | 前端 | 前端开发 |
| API 500/502/503 | 后端 | 后端开发 |
| API 404 | 后端 | 后端开发 |
| network timeout | 后端 | 后端开发 |
| fetch failed | 后端 | 后端开发 |

## 🐛 调试技巧

### 1. 运行单个测试文件
```bash
npx playwright test tests/e2e/01-homepage.spec.ts
```

### 2. 运行单个测试用例
```bash
npx playwright test -g "首页加载"
```

### 3. 看到浏览器运行
```bash
npm run test:e2e:headed
```

### 4. 查看 HTML 报告
```bash
npx playwright show-report
```

### 5. 截图和录影
失败时会自动保存在 `test-results/` 目录

---

## ⚠️ 注意事项

1. **测试前确保开发服务器已启动**
   - `npm run dev` 运行在 http://localhost:5173

2. **测试可能需要登录**
   - 后续会添加认证支持

3. **测试是只读的**
   - 不会修改数据库或持久化数据

4. **确认机制**
   - 所有修复操作都需要您确认后才执行
