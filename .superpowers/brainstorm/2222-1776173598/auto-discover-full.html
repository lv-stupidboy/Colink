│    dirs := scanSubdirs(pluginsDir)                                 │

│    // 生成all.go文件                                                 │

│  package all                                                        │
│                                                                     │
│  import (                                                           │
│  %s                                                                 │
│  )                                                                  │
│  `, imports)                                                        │
│  }                                                                  │
│                                                                     │
├────────────────────────────────────────────────────────────────────┤

│                                                                     │
│  package all                                                        │
│                                                                     │
│  import (                                                           │

│    _ "github.com/colink/isdp/internal/service/agent/plugins/open_code"    │
│    _ "github.com/colink/isdp/internal/service/agent/plugins/internal_agent" │
│  )                                                                  │
│                                                                     │
│  // 新增插件后，下次构建自动加入                                       │
│                                                                     │

新增插件：完全零改动

新增插件工作流
开发者只需：

1. 创建插件目录
   mkdir internal/service/agent/plugins/internal_agent

2. 实现插件文件
   ├── plugin.go    # init()注册
   ├── adapter.go   # AgentAdapter实现

3. 编译
   make build   # 或 go build（Makefile会自动执行genplugins）

就是这么简单！无需改动任何其他文件

数据流全景图

从插件到页面完整流程
启动时：

plugins/internal_agent/plugin.go
  └─→ init() 执行
      └─→ agent.RegisterPlugin(meta)
          └─→ 全局 registry["internal_agent"] = meta

API调用时：

GET /api/v1/base_agents/types
  └─→ BaseAgentService.GetTypes()
      └─→ AdapterRegistry.GetTypes()
          └─→ 遍历 registry 返回所有已注册类型

执行Agent时：

POST /api/v1/threads/xxx/agents
  └─→ ExecutionService.SpawnAgent()
      └─→ AdapterRegistry.GetAdapter(baseAgent)
          └─→ registry[type].Factory(baseAgent)
              └─→ NewInternalAgentAdapter(baseAgent)
                  └─→ 返回 AgentAdapter 实例
      └─→ adapter.Execute(ctx, req)
          └─→ InternalAgentAdapter 执行具体逻辑

内网部署：最小改动

内网Agent独立开发部署
方式1：直接放入Colink仓库

git clone https://github.com/colink/isdp.git
cd isdp/internal/service/agent/plugins
mkdir internal_agent
# 编写 plugin.go + adapter.go

make build  # 自动发现并编译

方式2：独立仓库 + go.mod replace

# go.mod
replace github.com/colink/isdp/plugins/internal_agent => ../internal-agent-plugin

# 内网仓库只需包含插件代码
internal-agent-plugin/
  ├── go.mod (module github.com/colink/isdp/plugins/internal_agent)
  └── plugin.go
  └── adapter.go

方式3：构建脚本复制

# 内网构建脚本
git clone https://github.com/colink/isdp.git
cp ../internal-agent/* isdp/internal/service/agent/plugins/internal_agent/
cd isdp && make build
