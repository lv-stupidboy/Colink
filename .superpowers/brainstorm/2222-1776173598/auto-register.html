编译时自动注册方案

约定目录结构 + init()自动注册

核心原理：Go init() 自动执行

插件包导入即自动注册
Go语言特性：包被导入时，init()函数自动执行

┌────────────────────────────────────────────────────────────────────┐
│  main.go                                                            │
│  ────────────────────────────────────────────────────────────────  │
│  import (                                                           │
│    // 导入插件目录（空导入，仅触发init）                              │
│    _ "github.com/colink/isdp/internal/service/agent/plugins/all"    │
│  )                                                                  │
│                                                                     │
│  或者更简单：                                                        │
│  import (                                                           │
│    _ "github.com/colink/isdp/internal/service/agent/plugins"        │
│  )                                                                  │
│                                                                     │
│  这会导入plugins目录下所有子包，触发各插件的init()注册                │
│                                                                     │
├────────────────────────────────────────────────────────────────────┤
│  plugins/all/all.go                                                 │
│  ────────────────────────────────────────────────────────────────  │
│  // 空导入所有插件包（触发各包的init）                                │
│  import (                                                           │
│    _ "github.com/colink/isdp/internal/service/agent/plugins/claude_code" │
│    _ "github.com/colink/isdp/internal/service/agent/plugins/open_code"    │
│  )                                                                  │
│                                                                     │
├────────────────────────────────────────────────────────────────────┤
│  plugins/claude_code/plugin.go                                      │
│  ────────────────────────────────────────────────────────────────  │
│  func init() {                                                      │
│    agent.RegisterPlugin(agent.PluginMeta{                          │
│      Type:        "claude_code",                                    │
│      Name:        "ClaudeCode",                                     │
│      Description: "Anthropic Claude CLI",                           │
│      Factory:     NewClaudeAdapter,                                 │
│      ConfigDir:   ".claude",                                        │
│      DefaultPath: "claude",                                         │
│    })                                                               │
│  }                                                                  │
│                                                                     │
├────────────────────────────────────────────────────────────────────┤
│  plugins/open_code/plugin.go                                        │
│  ────────────────────────────────────────────────────────────────  │
│  func init() {                                                      │
│    agent.RegisterPlugin(agent.PluginMeta{                          │
│      Type:        "open_code",                                      │
│      Name:        "OpenCode",                                       │
│      Description: "OpenCode CLI via ACP",                           │
│      Factory:     NewOpenCodeAdapter,                               │
│      ConfigDir:   ".opencode",                                      │
│      DefaultPath: "opencode",                                       │
│    })                                                               │
│  }                                                                  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘

关键点：
• agent.RegisterPlugin() 注册到全局变量
• 任何新插件只需在plugins目录下新增一个包
• 编译时自动发现、自动注册、零代码改动

目录结构约定

插件目录结构
internal/service/agent/
│
├── adapter_registry.go    # 全局注册中心 + RegisterPlugin函数
├── adapter.go             # AgentAdapter接口定义
├── types.go               # 公共类型定义
│
├── plugins/               # 插件目录（约定位置）
│   │
│   ├── all/               # 空导入所有插件（触发init）
│   │   └── all.go         # import _ "..." 所有子包
│   │
│   ├── claude_code/       # Claude插件包
│   │   ├── plugin.go      # init() + RegisterPlugin()
│   │   ├── adapter.go     # ClaudeAdapter实现
│   │   └── parser.go      # stream-json解析器
│   │
│   ├── open_code/         # OpenCode插件包
│   │   ├── plugin.go      # init() + RegisterPlugin()
│   │   ├── adapter.go     # OpenCodeAdapter实现（ACP）
│   │   └── acp_types.go   # ACP协议类型（可复用）
│   │
│   └── [internal_agent]/  # 内网插件包（新增时放这里）
│       ├── plugin.go      # init() + RegisterPlugin()
│       ├── adapter.go     # 内网Adapter实现
│       └── parser.go      # 如有特殊协议
│
├── orchestrator.go        # 编排层（不变）
├── execution_service.go   # 执行服务（改用Registry）
└── base_agent_service.go  # 基础Agent服务（改用Registry）

新增插件流程

零代码改动新增Agent
步骤1： 在plugins目录下创建新包
  internal/service/agent/plugins/internal_agent/
    ├── plugin.go    # init()注册
    ├── adapter.go   # 实现 AgentAdapter 接口
    └── parser.go    # 如需自定义解析

步骤2： 在plugins/all/all.go添加一行导入
  import (
    _ "github.com/colink/isdp/internal/service/agent/plugins/claude_code"
    _ "github.com/colink/isdp/internal/service/agent/plugins/open_code"
    _ "github.com/colink/isdp/internal/service/agent/plugins/internal_agent"  ← 新增
  )

步骤3： 编译
  go build ./cmd/server

完成！ 新插件自动注册，无需改动其他代码

自动发现方案（完全零改动）

使用generate工具自动生成all.go
方案： plugins/all/all.go 由 go generate 自动生成

// plugins/all/generate.go
//go:generate go run github.com/colink/isdp/tools/genplugins

// tools/genplugins/main.go
// 扫描 plugins/ 目录下所有子目录
// 生成 all.go 文件内容：
import (
  _ "github.com/colink/isdp/internal/service/agent/plugins/claude_code"
  _ "github.com/colink/isdp/internal/service/agent/plugins/open_code"
  // 任何新增插件自动加入
)

使用流程：
1. 新增插件目录 plugins/xxx/
2. 运行 go generate ./internal/service/agent/plugins/all
3. 编译 go build ./cmd/server

或者编译脚本中自动执行：
#!/bin/bash
go generate ./internal/service/agent/plugins/all
go build ./cmd/server

内网部署场景

场景： 开源Colink + 内网自研Agent

方案1：Fork + 新增插件包
  git clone colink
  cd internal/service/agent/plugins
  mkdir internal_agent
  # 实现插件...
  go generate ./plugins/all
  go build

方案2：独立插件仓库 + replace
  // go.mod
  replace github.com/colink/isdp/plugins => ../internal-plugins

  // 内网仓库只包含插件包
  internal-plugins/
    ├── go.mod
    └── plugins/
        └── internal_agent/
            ├── plugin.go
            ├── adapter.go

方案3：编译脚本自动合并
  // 构建时复制内网插件到plugins目录
  cp ../internal-agent/* ./internal/service/agent/plugins/internal_agent/
  go generate ./internal/service/agent/plugins/all
  go build ./cmd/server
