# 首页 Agent 团队与角色卡片设计

**日期：** 2026-04-16
**状态：** 设计确认，待实施

---

## 目标

在 Dashboard 首页新增两个卡片，展示 Agent 团队和 Agent 角色的基本信息，提供快捷入口跳转到对应管理页面。

---

## 需求背景

用户希望在首页能快速了解 Agent 团队和 Agent 角色的概况，无需每次都进入专门的列表页面查看。通过预览卡片形式，提供：
- 基本信息（数量统计）
- 快捷跳转入口

---

## 设计方案

### 方案选择：新增两个独立卡片

在现有统计卡片下方新增两个独立卡片：
- **Agent团队卡片**：团队数量 + 最近团队列表（含Agent数量）
- **Agent角色卡片**：角色总数 + 系统预置数 + 自定义数

**选择理由：**
1. 与现有 Dashboard 风格一致（现有是4个独立的统计卡片）
2. 职责单一，后续可独立扩展（如添加快捷操作按钮）
3. 并排显示更美观

---

## 详细设计

### 1. Agent团队卡片

**卡片布局：**
```
┌─────────────────────────────────────┐
│  🤝 Agent团队                    3 → │  ← 标题行：图标+名称+数量（点击跳转）
├─────────────────────────────────────┤
│  · 全栈开发团队 (4 Agents)          │  ← 最近团队列表（最多3个）
│  · 需求分析团队 (2 Agents)          │
│  · 测试团队 (3 Agents)              │
│                                     │
│  查看全部 →                         │  ← 底部链接
└─────────────────────────────────────┘
```

**数据来源：**
- 调用 `api.workflows.list()` 获取团队列表
- 显示前3个团队（按 updatedAt 降序）
- 每个团队显示：名称 + `agentIds.length`

**交互：**
- 点击标题区域跳转到 `/workflow` 页面
- 点击"查看全部"跳转到 `/workflow` 页面

**空状态处理：**
- 当团队数为0时，显示"暂无团队，点击创建"
- 点击跳转到 `/workflow` 并打开创建弹窗

---

### 2. Agent角色卡片

**卡片布局：**
```
┌─────────────────────────────────────┐
│  🤖 Agent角色                    5 → │  ← 标题行：图标+名称+总数（点击跳转）
├─────────────────────────────────────┤
│  系统预置: 3                         │  ← 统计信息
│  自定义: 2                           │
│                                     │
│  查看全部 →                         │  ← 底部链接
└─────────────────────────────────────┘
```

**数据来源：**
- 调用 `api.agents.list()` 获取角色列表
- 计算总数、系统预置数量（`isSystem === true`）、自定义数量（`isSystem === false`）

**交互：**
- 点击标题区域跳转到 `/agents` 页面
- 点击"查看全部"跳转到 `/agents` 页面

---

### 3. 页面布局位置

**在 Dashboard 中的位置：**
```
┌────────────────────────────────────────────────────────────┐
│  Dashboard                                                 │
│  Welcome to Colink                                         │
├────────────────────────────────────────────────────────────┤
│  [项目总数] [进行中任务] [已完成任务] [待审查]               │  ← 现有统计卡片（Row + 4 Col）
├────────────────────────────────────────────────────────────┤
│  [Agent团队卡片]  [Agent角色卡片]                           │  ← 新增：Row + 2 Col（lg=12）
├────────────────────────────────────────────────────────────┤
│  活跃任务                                                   │  ← 现有活跃任务区域
│  ...                                                       │
├────────────────────────────────────────────────────────────┤
│  最近项目                                                   │  ← 现有最近项目区域
│  ...                                                       │
└────────────────────────────────────────────────────────────┘
```

**布局细节：**
- 两个新卡片放在统计卡片和活跃任务之间
- 使用 `Row gutter={[16, 16]}`
- 每个卡片占 `Col xs={24} lg={12}`（小屏单列，大屏双列）
- 卡片高度适中，与现有统计卡片风格保持一致

---

### 4. 样式规范

**遵循项目深色模式适配规则：**
- 使用 CSS 变量：`var(--bg-container)`、`var(--text-primary)`、`var(--border-color)`
- 禁止硬编码颜色值
- 参考现有统计卡片的样式实现

**图标选择：**
- Agent团队：`TeamOutlined` 或 `UsergroupOutlined`
- Agent角色：`RobotOutlined`

---

### 5. 技术实现要点

**修改文件：**
- `web/src/pages/Dashboard/index.tsx` — 新增两个卡片组件和数据加载逻辑

**API 调用：**
- 在现有 `loadDashboardData` 函数中并行添加：
  - `api.workflows.list()` — 获取团队列表
  - `api.agents.list()` — 获取角色列表

**状态管理：**
- 新增 state：`workflows`、`agentConfigs`、`loadingAgents`
- 在 `useEffect` 中并行加载所有数据

---

## 边界情况

| 场景 | 处理方式 |
|------|----------|
| 团队数为0 | 显示"暂无团队"，点击跳转创建 |
| 角色数为0 | 显示"暂无角色"，点击跳转创建 |
| API加载失败 | 卡片显示加载失败状态，不影响其他卡片 |
| 数据量大 | 团队列表只显示前3个，避免卡片过长 |

6. 小屏幕响应式布局正常（单列显示）
