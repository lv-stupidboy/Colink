# ISDP Windows 安装器设计文档

## 概述

### 背景

ISDP即将进入内测阶段，需要制作Windows平台的安装包，让用户能够方便地安装和使用ISDP智能开发平台。

### 目标

1. 提供图形化的安装向导，引导用户完成安装
2. 自动检测并安装运行依赖（Node.js、Git、Claude CLI、OpenCode）
3. 在安装过程中完成数据库等系统配置
4. 安装完成后用户可直接使用平台，无需额外配置

### 目标用户

- Windows系统用户
- 本机安装前后端和依赖软件
- 数据库使用远程服务
- Redis和Docker暂不需要

### 核心组件

| 组件 | 说明 |
|------|------|
| **安装器** | Electron + React GUI，负责安装向导和依赖管理 |
| **应用本体** | 后端exe + 前端静态文件，约100MB |
| **启动器** | 安装后生成的独立程序（ISDP-Launcher.exe），提供系统托盘管理服务启停 |
| **依赖工具** | 可选安装Claude CLI / OpenCode |

### 启动器架构说明

启动器是安装完成后生成的独立可执行文件，与安装器分离：

1. **安装阶段**：安装器（ISDP-Setup.exe）负责安装流程
2. **安装完成**：安装器在目标目录生成 `ISDP-Launcher.exe` 作为启动器
3. **日常使用**：用户通过桌面快捷方式启动 `ISDP-Launcher.exe`，该程序提供系统托盘功能

启动器本质上是安装器的精简版，复用Electron的托盘模块，但不需要安装向导的UI组件。

## 二、安装流程设计

### 流程图

```
[欢迎] → [目录选择] → [依赖检测] → [模式选择] → [系统配置] → [安装] → [完成]
```

### Step 1: 欢迎

- 展示ISDP品牌和欢迎信息
- 说明安装向导将完成的任务
- 提供"开始安装"按钮

### Step 2: 安装目录选择

- 用户选择安装位置
- 显示所需空间和可用空间
- 默认路径：`C:\Program Files\ISDP`

### Step 3: 依赖检测

检测以下依赖项：

| 依赖 | 必需 | 检测命令 |
|------|------|----------|
| Node.js | 是 | `node --version` |
| Git | 是 | `git --version` |
| Claude CLI | 否 | `claude --version` |
| OpenCode | 否 | `opencode --version` |

显示检测结果，标注缺失项数量。

### Step 4: 模式选择

仅当有缺失依赖时显示此步骤。提供三种选择：

1. **自动安装（推荐）**：安装器自动下载并安装缺失依赖
2. **手动安装**：用户自行安装，完成后继续
3. **跳过安装**：暂不安装，后续在平台配置

### Step 5: 系统配置

通用的配置页面，支持可折叠的配置区块：

- **数据库配置**（必填，默认展开）
  - 数据库地址
  - 端口
  - 数据库名
  - 用户名/密码
  - 测试连接按钮

- **高级设置**（可选，默认折叠）
  - 预留扩展空间
  - 如HTTP代理等

设计要点：
- 可折叠的配置区块，便于后续扩展新配置项
- 数据库连接测试功能

### 数据库连接测试机制

由于安装阶段后端服务尚未启动，安装器需要内置独立的数据库连接测试模块：

1. **技术方案**：使用 Node.js 的 `mysql2` 库进行连接测试
2. **测试流程**：
   - 用户点击"测试连接"按钮
   - 安装器主进程调用 `mysql2.createConnection()` 尝试连接
   - 连接成功显示绿色勾号，失败显示错误信息
3. **依赖**：mysql2 作为安装器的依赖打包，无需用户额外安装

### Step 6: 安装进度

显示安装进度：

1. 复制文件
2. 安装Claude CLI（如选择自动安装）
3. 安装OpenCode（如选择自动安装）
4. 生成配置文件

### Step 7: 完成

- 显示安装成功
- 显示安装位置
- 可选项：
  - 创建桌面快捷方式
  - 立即启动ISDP

## 三、依赖检测与安装

### 依赖定义

```typescript
interface Dependency {
  name: string;           // 显示名称
  key: string;            // 标识符
  required: boolean;      // 是否必需
  checkCommand: string;   // 检测命令
  versionRegex: RegExp;   // 版本提取正则
  minVersion?: string;    // 最低版本要求
}
```

### 安装方式

| 依赖 | 安装方式 | 说明 |
|------|----------|------|
| **Node.js** | 下载官方msi静默安装 | 检测到已有则跳过 |
| **Git** | 下载官方exe静默安装 | Windows必需，Claude CLI依赖 |
| **Claude CLI** | `npm install -g @anthropic-ai/claude-cli` | 需要先有Node.js |
| **OpenCode** | `npm install -g @anthropic-ai/opencode` | 需要先有Node.js |

### 容错机制

- 必需依赖（Node.js、Git）失败 → 提示用户，允许重试或手动安装
- 可选依赖（Claude CLI、OpenCode）失败 → 警告用户，继续安装，后续可在平台配置
- 已安装的依赖自动跳过

## 四、启动器设计

### 功能

安装完成后，用户通过启动器使用ISDP：

1. 系统托盘图标
2. 右键菜单功能：
   - 打开控制台（打开浏览器 localhost:3000）
   - 服务状态显示
   - 查看日志
   - 重启服务
   - 退出

### 启动流程

```
双击桌面快捷方式
        ↓
    系统托盘出现ISDP图标
        ↓
    启动后端服务
        ↓
    自动打开浏览器 http://localhost:3000

    用户开始使用平台
```

## 八、实现要点

### 主进程职责

1. 窗口管理：创建无边框窗口，自定义标题栏
2. 安装协调：调用各安装服务，更新进度
3. 文件操作：复制应用文件到目标目录
4. 配置生成：根据用户输入生成config.yaml

### 渲染进程职责

1. UI渲染：安装向导各页面
2. 用户交互：收集用户输入
3. 进度展示：实时显示安装进度
4. IPC通信：与主进程交互

### 安全考虑

1. 不在安装包中包含敏感信息
2. 数据库密码在安装时由用户输入
3. 配置文件权限控制

### 卸载策略

卸载时的清理策略：

| 内容 | 是否删除 | 说明 |
|------|----------|------|
| 应用程序文件 | 是 | isdp-server.exe, web/, launcher |
| 配置文件 | 询问用户 | config.yaml 可选择保留以便重装 |
| 日志文件 | 是 | logs/ 目录 |
| 数据库数据 | 否 | 远程数据库不受影响 |
| 用户数据 | 询问用户 | agent-assets/, repos/ 等用户工作目录 |

### 回滚策略

安装失败时的回滚机制：

1. **文件复制失败**：删除已复制的文件，恢复原始状态
2. **依赖安装失败**：
   - 必需依赖失败：提示用户，提供"重试"和"手动安装"选项
   - 可选依赖失败：警告用户，允许继续安装
3. **配置生成失败**：提示错误，允许用户修改配置后重试
4. **整体回滚**：提供"取消安装"选项，清理所有已安装内容
