## 需求概述

在 installer-tauri 的 Launcher 中修改「系统配置」按钮行为：
- **现状**：点击后打开配置文件所在文件夹
- **目标**：点击后打开弹窗，直接编辑 config.yaml 内容

## 用户决策

| 决策点 | 选择 | 说明 |
|--------|------|------|
| 编辑体验 | 重启提醒 | 保存后弹窗提示「配置已更新，重启服务生效」，提供重启按钮 |
| 配置可见性 | 完全展示 | yaml 文本直接显示，无敏感字段隐藏 |
| 编辑方式 | yaml 文本编辑 | 直接编辑原始 yaml 内容，非结构化字段编辑 |
| 编辑器 | 语法高亮 | Monaco Editor 或 CodeMirror，提供 yaml 语法高亮 |
| 触发位置 | 现有按钮 | 修改现有「系统配置」按钮行为 |

## 技术方案

### 1. 修改触发行为

**文件**：`src-tauri/src/commands/launcher.rs` 或前端 Launcher 组件

**改动**：
- 移除「打开文件夹」逻辑
- 添加「打开配置弹窗」逻辑

### 2. 配置弹窗组件

**新增文件**：`src/components/ConfigEditorModal.tsx`（或类似命名）

**功能**：
- Modal 弹窗，标题「系统配置」
- 内容：yaml 编辑器（Monaco/CodeMirror）
- 底部按钮：「取消」「保存」

### 3. yaml 编辑器选型

**方案 A：Monaco Editor**
- 优点：VS Code 同款，功能强大，内置 yaml 支持
- 缺点：体积较大（~2MB）

**方案 B：CodeMirror 6**
- 优点：轻量、模块化、性能好
- 缺点：需要手动配置 yaml mode

**推荐**：CodeMirror 6（installer-tauri 已使用，保持一致性）

### 4. 保存流程

```
用户点击保存
  ↓
验证 yaml 格式（可选，暂不强制）
  ↓
写入 config.yaml 文件
  ↓
弹窗提示「配置已更新，重启服务生效」
  ↓
用户选择：
  - 点击「重启」→ 调用重启服务逻辑
  - 点击「关闭」→ 关闭提示弹窗
```

### 5. Tauri Command

**新增**：`save_system_config(content: String) -> Result<(), String>`

**实现**：

## 已确认问题

1. **yaml 格式验证**：保存前强制验证 yaml 格式，错误时阻止保存并提示
   - 实现：前端使用 yaml 解析库验证，错误时显示错误信息

2. **重启服务逻辑**：
   - 使用「停止服务 → 启动服务」流程
   - 停止服务需要遵循现有校验逻辑：检查是否存在活跃中的 agent 会话
   - 如果有活跃会话，弹出确认弹窗（与现有停止服务行为一致）

3. **编辑器尺寸**：
   - Modal 固定高度 600px + 内部滚动

| UI | Modal 弹窗、编辑器、按钮 | 配置分组折叠、字段级表单 |

## 成功标准

1. 点击「系统配置」按钮打开弹窗（不打开文件夹）
2. 弹窗展示 config.yaml 内容，支持语法高亮
3. 用户可编辑内容并保存
4. 保存后提示重启服务
5. 重启按钮可正常重启服务
