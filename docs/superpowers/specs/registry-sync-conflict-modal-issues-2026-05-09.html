问题1：同名不同路径 Skill 选中同步问题

当有多个同名不同路径的 Skill 时，点击选中其中一个，其他同名的 Skill 会被同时选中/取消选中。

❌ 当前问题状态
同步冲突处理
brainstorm 来自: term
路径: skills/brainstorm/SKILL.md
远程描述: 需求探索技能...

brainstorm 冲突
路径: .claude/skills/brainstorm/SKILL.md
远程描述: 创意头脑风暴...
⚠️ 被同步选中！
根本原因：
  • rowKey="name" - React 使用 name 作为行唯一标识
  • conflictChoices[skill.name] - 状态 key 使用 name 而非唯一标识
  • 同名不同路径的 Skill 共享同一个选择状态

问题2：新建 Skill 联邦源导入不显示目录路径

点击新建 Skill 选择联邦源导入时，扫描弹窗只显示 Skill 名称，不显示目录路径，用户无法区分同名不同路径的 Skill。

brainstorm
创意头脑风暴，探索多种可能...
❌ 与上面同名，用户无法区分
根本原因：
  • 扫描弹窗只渲染 skill.name 和 skill.description
  • 没有渲染 skill.path 字段
// SkillLibrary/index.tsx:1372-1399 <List dataSource={scanResult.skills} renderItem={(skill) => ( <List.Item> <Checkbox ...> <div> <Text strong>{skill.name}</Text> {skill.existsLocally && ...} <br /> // 缺少 skill.path 显示 <Text type="secondary">{skill.description || '暂无描述'}</Text> </div> </Checkbox> </List.Item> )} />
