# 首页排版优化需求规格

## 需求概述

对首页（Dashboard）展示排版进行优化，调整布局结构并新增最近任务展示功能。

## 当前状态

### 现有布局结构
```
┌─────────────────────────────────────────────────────┐
│ Hero Section（居中标题 + 开始新任务按钮）              │
├────────────────────────┬────────────────────────────┤
│ 项目与任务              │ 团队与资产                  │
│ - 统计卡片（项目/进行中）│ - 统计卡片（团队/角色配置）  │
│ - 活跃任务              │ - Agent团队卡片列表         │
│ - 最近项目              │                            │
└────────────────────────┴────────────────────────────┘
```

### 关键文件
- 前端：`web/src/pages/Dashboard/index.tsx`
- 后端：`internal/api/dashboard_handler.go`

## 需求详情

### 需求1：布局调整为上下排布

**变更内容**：
- 将"项目与任务"和"团队与资产"从左右并排布局改为上下垂直排布
- 两个区块各占一行，宽度100%

**目标布局**：
```
┌─────────────────────────────────────────────────────┐
│ Hero Section（居中标题 + 开始新任务按钮）              │
├─────────────────────────────────────────────────────┤
│ 项目与任务                                           │
│ - 统计卡片（项目/进行中）                             │
│ - 活跃任务                                           │
│ - 最近任务（新增）                                    │
│ - 最近项目                                           │
├─────────────────────────────────────────────────────┤
│ 团队与资产                                           │
│ - 统计卡片（团队/角色配置）                           │
│ - Agent团队卡片列表                                  │
└─────────────────────────────────────────────────────┘
```

### 需求2：新增"最近任务"展示

**功能描述**：
- 在"活跃任务"下方新增"最近任务"区域
- 按任务更新时间（updatedAt）倒序排列
- 展示最近更新的3个任务

**展示信息**：
- 任务名称
- 所属项目名称
- 所属团队名称（即 Agent 团队，对应 workflow_templates.name）
- 更新时间
- 任务状态（标签形式）

**交互**：
- 点击任务卡片跳转到任务详情页

## 技术实现方案

### 前端变更

**文件**：`web/src/pages/Dashboard/index.tsx`

1. **布局调整**：
   - 移除 `Row gutter={20}` 包裹的两个 `Col`
   - 改为两个独立的 `Card` 区块垂直排列
   - 每个 `Card` 占满整行宽度

2. **新增最近任务组件**：
   ```tsx
   // 最近任务卡片组件
   const RecentThreadCard: React.FC<{
     thread: RecentThreadInfo;
     onClick: () => void;
   }> = ({ thread, onClick }) => (
     // 展示任务名称、项目、团队、更新时间、状态
   );
   ```

3. **新增状态和数据加载**：
   ```tsx
   const [recentThreads, setRecentThreads] = useState<RecentThreadInfo[]>([]);
   
   // 加载最近任务
   const loadRecentThreads = useCallback(async () => {
     const threads = await api.dashboard.getRecentThreads();
     setRecentThreads(threads);
   }, []);
   ```

### 后端变更

**文件**：`internal/api/dashboard_handler.go`

1. **新增 API 接口**：
   - 路由：`GET /api/v1/dashboard/recent-threads`
   - 功能：获取最近更新的任务列表

2. **新增数据结构**：
   ```go
   // RecentThreadInfo 最近任务信息
   type RecentThreadInfo struct {
       ID                 string   `json:"id"`
       ProjectID          string   `json:"projectId"`
       Name               string   `json:"name"`
       Status             string   `json:"status"`
       CurrentPhase       string   `json:"currentPhase"`
       WorkflowTemplateID string   `json:"workflowTemplateId"`
       UpdatedAt          string   `json:"updatedAt"`
       ProjectName        string   `json:"projectName"`
       TeamName           string   `json:"teamName"`            // 所属团队名称（Agent团队，来自workflow_templates.name）
   }
   ```

3. **新增查询方法**：
   ```go
   // GetRecentThreads 获取最近更新的任务列表
   func (h *DashboardHandler) GetRecentThreads(c *gin.Context) {
       // 按 updatedAt DESC 排序，LIMIT 3
   }

   // queryRecentThreads 查询最近更新的任务
   func (h *DashboardHandler) queryRecentThreads(ctx context.Context) []RecentThreadInfo {
       query := `
           SELECT t.id, t.project_id, t.name, t.status, t.current_phase,
                  t.workflow_template_id, t.updated_at,
                  p.name as project_name, wt.name as team_name
           FROM threads t
           LEFT JOIN projects p ON t.project_id = p.id
           LEFT JOIN workflow_templates wt ON t.workflow_template_id = wt.id
           ORDER BY t.updated_at DESC
           LIMIT 3
       `
       // ...
   }
   ```

4. **注册路由**：
   ```go
   dashboard.GET("/recent-threads", h.GetRecentThreads)
   ```
