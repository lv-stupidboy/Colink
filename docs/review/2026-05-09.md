## 主要问题 (已修复)

### 1. 测试用例 BT-04 使用错误的 API 路径 ✅ 已修复

**位置**: `auto-test/e2e/api/backend-api.spec.ts:109`

**问题描述**:  
测试用例请求 `/api/v1/threads` 期望返回所有 threads 的 JSON 列表，但实际返回了 HTML（前端页面）。

**根本原因**:  
后端 `thread_handler.go` 的路由设计中，**没有** `threads.GET("/")` 路由来列出所有 threads：

```go
// thread_handler.go RegisterRoutes
threads.GET("/:id", h.Get)
threads.GET("/project/:projectId", h.ListByProject)  // 按项目列出
threads.POST("/project/:projectId", h.Create)
```

后端设计要求通过 `/api/v1/threads/project/:projectId` 按 project ID 获取 threads 列表。

**修复方案**:  
修改测试用例，改为：
1. 先获取项目列表 `/api/v1/projects`
2. 使用第一个项目的 ID 请求 `/api/v1/threads/project/:projectId`
3. 如果没有项目，跳过测试

**修复验证**: E2E 测试全部通过（5 passed）
